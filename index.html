Battle Engine

Turn-Based Rule Engine

Rock-paper-scissors style resolution with critical hits and a weighted AI opponent. Fully server-side — client just picks an action.

⚔️
Attack
Full damage. Beats Weak Attack. Loses to Block.
🛡️
Block
Absorbs Attack, deals 30% counter. Loses to Weak Attack.
💨
Weak Attack
60% damage that bypasses Block. Loses to Attack.
Outcome Matrix
Player Action Opponent Action Result Damage Dealt
⚔️ Attack 💨 Weak Attack Win 100% DMG to opponent
🛡️ Block ⚔️ Attack Win 0 taken + 30% counter
💨 Weak Attack 🛡️ Block Win 60% bypasses block
⚔️ Attack ⚔️ Attack Draw Both take 100% DMG
🛡️ Block 🛡️ Block Draw No damage either side
⚔️ Attack 🛡️ Block Lose 30% counter damage taken
🎲 Critical Hit: 15% chance on player wins → 1.5× damage multiplier 15% · 1.5×

Battle Flow
1
startBattle
Validate squad exists. Pick random opponent (not self). Snapshot both squads' stats. Create battle record. Set KV lock.
BATTLE_LOCK[studentId] = battleId status: IN_PROGRESS
2
submitAction (per turn)
Player picks ATTACK / BLOCK / WEAK_ATTACK. AI picks opponent action. ActionResolver computes damage. Both sides' HP updated.
Max 5 DB ops per turn Crit roll server-side
3
End Condition Check
After each turn: check HP thresholds or turn limit. If ended, apply global health damage and mark battle status.
status: WIN | LOSE | DRAW
✓
claimBattleReward
Write XP to user_xp. Write points to transaction_history. Clear KV lock. Battle is now claimable only once.
XP: user_xp table Points: transaction_history KV lock cleared

Technical Challenges

Key Problems & Solutions

🔒
Concurrent Battle Prevention
Student double-taps "Start Battle" and two battles spawn simultaneously, corrupting HP state.
Solution
BATTLE_LOCK KV namespace. Set BATTLE_LOCK[studentId] = battleId at start. Check before every action. Clear on battle end.
🎲
No-Duplicate Egg Characters
Egg system must never award a character the student already owns.
Solution
EggService queries owned characterIds first. Random picks from the remainder set. Fallback to bonus points if all 6 owned.
📸
Async Opponent Consistency
Opponent student changes their squad mid-battle, changing the difficulty mid-fight.
Solution
Snapshot opponent squad stats into battle.defenderHpSnapshot at startBattle. Live changes never touch an ongoing battle.
⭐
XP vs Points Isolation
Two currency systems that must never interfere. XP only levels, Points only spend.
Solution
Completely separate tables (user_xp vs transaction_history) and separate services. No shared code paths between the two.
⚡
D1 CPU Time Limits
Cloudflare Workers have ~50ms CPU budget. Complex battle turns risk timeout.
Solution
Max 5 DB operations per submitAction call. Use JOINs, not N+1. Battle engine is pure TypeScript — zero I/O in resolution logic.
🛡️
Shield Item Timing
Shield must block damage from async battles that happen while the student is offline.
Solution
UserHealthService.applyDamage() reads shieldActive flag before reducing HP. Shield consumed after battle ends. Potion uses timestamp expiry check.
